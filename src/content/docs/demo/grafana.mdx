<Aside type="note">
  Terraform code:
  [terraform-aws-lambda-managed-instance](https://github.com/jajera/terraform-aws-lambda-managed-instance).
  On first boot, EC2 `user_data` writes datasource YAML and dashboard JSON under
  `/opt/obs/grafana/`. Terraform renders `templates/user_data.sh.tftpl` and injects
  `templates/dashboards/*.json` as base64 so the instance files match the repo.
</Aside>

Grafana runs alongside Loki on the same EC2 host via Docker Compose. A public ALB forwards HTTP port 80 to Grafana port 3000. The Loki datasource and WAF dashboards are provisioned from those files at container startup — no manual setup required.

## Access

```bash frame="terminal"
terraform output grafana_url
# Prints: http://<alb-dns-name>
```

Open the URL in a browser. Default credentials: **admin / admin**. Grafana prompts for a password change on first login.

<Aside type="caution">
  The ALB security group restricts ingress to your deployer's current public IP by default (via
  `data.http` `checkip.amazonaws.com`). If your IP changes, update `alb_ingress_cidrs` in
  `terraform.tfvars` and run `terraform apply` to refresh the security group rule.
</Aside>

## Provisioned datasource

Grafana automatically connects to Loki at `http://loki:3100` using Docker Compose internal DNS. The datasource is set as the default and requires no manual configuration:

```yaml title="Provisioning excerpt (written by user_data.sh.tftpl)"
apiVersion: 1
datasources:
  - name: Loki
    type: loki
    uid: loki
    url: http://loki:3100
    access: proxy
    isDefault: true
```

## WAF dashboards (folder **WAF**)

**WAF Logs** — one **Logs** panel on `{source="waf"}` for full JSON lines (newest first).

**WAF overview** — common WAFv2-style views on the same stream:

- Log volume per minute
- Time series by **`action`** (ALLOW, BLOCK, COUNT, CAPTCHA, …)
- Time series by **`terminatingRuleId`**
- Top client IPs via `| json clientIp="httpRequest.clientIp"`
- HTTP methods via `| json method="httpRequest.httpMethod"`
- **Logs** panel filtered to **`action="BLOCK"`** for triage

Panels assume AWS WAFv2 JSON body shape. If a chart is empty, confirm your objects match that schema or adjust the LogQL `json` field paths in the dashboard (Explore is the quickest place to experiment).

## LogQL queries

Query WAF log streams in **Explore** (left sidebar → compass icon):

```text title="All WAF log lines"
{source="waf"}
```

```text title="BLOCK actions only"
{source="waf"} | json | action = "BLOCK"
```

```text title="Client IP filter"
{source="waf"} | json | line_format "{{.httpRequest.clientIp}} {{.action}} {{.httpRequest.uri}}"
```

```text title="Request count by action over time"
sum by (action) (count_over_time({source="waf"} | json [5m]))
```

## ALB health check

The ALB target group health check hits `GET /api/health` on port 3000. Grafana returns `200 OK` once the container is fully initialised. The EC2 Docker Compose `healthcheck` on the Loki service gates Grafana startup — expect 2–4 minutes from `terraform apply` completion before Grafana is reachable via the ALB.

## Restarting the stack on EC2

If you need to restart Compose manually (e.g. after updating config files):

```bash frame="terminal"
# Connect via SSM
aws ssm start-session --target "$(terraform output -raw obs_instance_id)"

# Inside the session
cd /opt/obs
docker compose down
docker compose up -d
```
